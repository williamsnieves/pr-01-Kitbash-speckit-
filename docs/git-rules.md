# Git Rules

## Branches
- `spec/001-*` for spec refinements.
- `plan/001-*` for plan refinements.
- `tasks/001-*` for task breakdown updates.
- `impl/001-*` for implementation phases.
- `fix/*` for corrections.

## Commits
- Follow Conventional Commits.
- Example: `docs(spec): refine spec 001 export scope`
- Example: `feat(scene): add renderer bootstrap`

## PR titles
- Use Conventional Commits format.

## Push discipline
- Run relevant checks before pushing.
- Keep PRs focused on one phase or slice.
